## 张力清单（T1–T8）

### T1 图文冲突：立场已裁（报告冲突），但"检测机制"是两回事

DR 修订后与教程 15 §4 立场一致：冲突 = 数据缺陷，报告而非择一。坑：**系统
怎么知道冲突存在？** 两个层次要在 SPEC 里分开：(a) **机械可检**——hotspot
编号集合 diff（描述 vs XML 声明），入库期确定性检测，冲突则该图降级；
(b) **语义冲突**（图说安全阀、文说排气管）——玩具切片**不实现自动检测**，
验收口径退一档：陷阱题上系统**不择一硬答**（拒答或同时呈现两个来源）即算过。
把 (b) 说成"能检测冲突"就是造假声明，EVIDENCE.md 红线。

### T2 合成 SVG 的文字可确定性提取——既是免费质检器，也是高估陷阱

DR 的"lxml 剥离 SVG `<text>` 节点做白名单锚点"比教程 15 的 hotspot 互验更强，
两者都做（成本几乎为零）。但硬币反面：**能从源码提取全部文字的图，对 VLM
是最友好的图**——本日测出的描述质量**不可外推**到真实扫描图/CGM（INV-7
玩具规模声明要点名这一条）。面试口径："我的质检器利用了合成数据的特权，
真实场景这道闸要换成人工抽检。"

### T3 栅格化管线是新依赖，选轻的

SVG→PNG 需要渲染器。DR 提的 headless browser 是重炮；轻量选项 cairosvg
（纯 Python 栈）或 resvg（二进制）。坑：**字体**——自绘 SVG 若用中文标注，
渲染环境缺字体 = 方框 = VLM 读不到，且 CI 环境与本机字体不同会导致图的
checksum 不稳定。**裁法**：标注用 ASCII（零件号/hotspot 编号本来就是），
图题可英文；渲染分辨率作为配置常量（见 T5），产物 PNG 进 ingest 管线、
checksum 对 PNG 算（渲染确定性问题就不影响绑定）。

### T4 VLM 供应商是本日最大的外部未知数

现栈的生成模型是文本模型，**没有已验证的 vision 通道**。选项：(a) 现有
provider 若有 VLM 端点，沿用（密钥/计费已在围栏内）；(b) 新增 provider
（新 key、新费用面、demo_guard 要扩展覆盖）。SPEC 决策层必须裁：用哪家、
预算多少、二次看图是否本日实现（建议：**不实现**，只做入库期描述——二次
看图连同 agent 复核整体记 Roadmap，教程 15 §3 的编排叙事面试口述即可）。
无论选哪家：温度 0、结构化输出、每图独立调用（DR 陷阱一）。

### T5 最低可用分辨率要实测，不拍脑袋

token 成本随分辨率平方级涨（DR 的 tiling/动态分辨率机制）。做法照 DR 选型
判据：**自建 10 图小测集**（不同分辨率 × hotspot 读取正确率），找"编号全对
的最低档"定为配置常量，测试数据进 notes。这是 INV-5 在多模态的第一次应用：
分辨率这个数字也要有出处。

### T6 图描述 chunk 挤占正文召回：43 chunk 的小池子里更敏感

语料只有 43 个 chunk，加 2–3 个术语密度极高的图描述 chunk，占比 5%+，挤占
效应比大语料更明显。DR 的父子分块方案玩具规模不值得上（记 Roadmap）。
**最小防线**：图描述 chunk 带独立 type 标记；旧 golden 全量回归跑一遍
（Day 4 harness 现成），旧指标掉了就给图 chunk 降权或隔离池。回归结果
不论好坏进 README（诚实纪律）。

### T7 引用与 trace 的 schema 要扩展，机器可读性资产同步

图引用 = ICN id + 描述 checksum + 命中字段，意味着 answer trace JSON、
citation 渲染、EVIDENCE.md、llms.txt 都要加图这一类证据。坑：Day 9 的
"陌生 agent 5 分钟定位任意数字"验收对图引用同样成立——图的复跑命令
（重新生成描述并 diff）也要写进 EVIDENCE.md，否则证据链在图这里断链。

### T8 评估集 n=8–10，报数方式提前定好

三类题（在图/在文/冲突陷阱）各 3 个左右，任何百分比都是大颗粒（1 题 =
10+ 个百分点）。按 Day 4 惯例：n<3 的类目斜体标"指示性"；总表报**分数
（k/n）而不是百分比**，防止"33% 提升"这种小样本幻觉数字进 README。
标注（哪个字段答案、该不该拒）仍是人做——检索评估红线不因模态改变。

## 必须吃透的点（must-master）

1. **图像 token 化两种机制**（DR §核心技术）：固定网格 tiling（85+170×n）
   vs 原生动态分辨率（28 px 倍数 + 14×14 patch + MRoPE）——成本模型和
   "为什么高分辨率贵得不成比例"，能画图讲。
2. **结构化输出的原理**：解码期对不合 schema 的 logits 做掩码 ⇒ 输出 100%
   合法 JSON——和"prompt 里求它输出 JSON"是两个可靠性等级；枚举（Enum）
   封闭候选集是压幻觉的第一杠杆。
3. **三条路线的取舍**（教程 15 §0 / DR 对比表）：describe-then-index 的
   "描述边界 = 检索边界"缺陷、CLIP 全局池化对线框图的表征崩溃、ColPali
   的 MaxSim 与存储代价——Q1/Q2 级面试题，脱稿。
4. **描述漂移与 checksum 绑定**：缓存失效类比、事件驱动重生成（本项目用
   同管线重建替代，规模小）——与 Day 4 manifest 是同一条纪律的两个应用。
5. **二次看图的预算护栏设计**（即使本日不实现）：限次/降分辨率/trace——
   与 Day 10 demo_guard 同构，面试时把两者串成"成本 fail-closed"叙事。
6. **合成数据特权声明**（T2）：哪些结论可外推（管线结构、schema 方法、
   护栏设计）、哪些不可（描述质量数字、分辨率阈值）——诚实分层的多模态版。

## 盲区四象限（Anthropic "Finding Your Unknowns"）

- **知道自己知道**：引用/拒答/trace 框架（Day 5）、陷阱题构造方法（Day 8）、
  费用拦截模式（Day 10）、L3 ICN 存在性检查（Day 2）——全部直接复用。
- **知道自己不知道**：所选 VLM 对合成线框图小字的实际读取精度（T5 小测集
  回答）；每图 token 成本实数；描述生成的端到端耗时（入库期，不敏感但要记）。
- **不知道自己知道**：Day 7 agent 的工具契约经验 = VLM 结构化输出的同款
  纪律（schema 即接口，宽进严出）；Day 8 的"评估器自身要被评估"直接适用
  于"描述质检器自身要抽查"。
- **不知道自己不知道**（红队重点）：所选 provider vision API 的实际行为
  （分辨率上限、静默压缩、结构化输出支持度——文档与实测常不一致）；SVG
  渲染的跨环境确定性（T3 只缓解未消除）；图描述进 BM25 路后对 analyzer
  的影响（描述里的下划线/驼峰 ID 被分词器切碎的老问题，Day 3 教训在新
  文类上复发）。
